Addressing and opening a file by clicking a hyperlink

6 Feb. 2011, rev. 14 March 2011

Introduction

In Part 2 a URL is only mentioned in passing, but no application model is given about what happens when you click on some word(s) or image, behind which a hyperlink is hidden, or a straightforward hyperlink.

Is this what you have been waiting for? Probably not, but it is a nice exercise, using already known templates, and showing that even a rather esoteric happening like this can be modelled with our templates.

In order not to mess up the template diagrams I have, by exception, not shown the cardinalities for the instances of subtypes of ClassOfRelationship.

Model

This is quite a complex model, some of the top part of the model "URL of a document" has been re-used. The bottom part of this model at hand is for the most a re-use of the model in the topic "Activity Models", the fourth model.

Whenever you click on the words 'click here' (an event) then the state change from "ending" to "beginning", i.e. is starts the applicable application (in this example presenting the selected PDF image).
